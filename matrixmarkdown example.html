Matrices as Markdown Tables

In addition to using commands like tabulate, estimates table, etc. to produce native Markdown tables (without code-fenced monospace), it is easy to write a function that will display any matrix as a Markdown table.

First, we will run an estimation command, which produces (at least) two matrices, e(b) and e(V).

. clear all

. sysuse auto
(1978 Automobile Data)

. quietly regress mpg price weight i.foreign

Now we can display the coefficient and precision matrices (no code fences).

. mata: matrixmarkdown(“e(b)”)

price weight 0b.foreign 1.foreign _cons
.0000566 -.0067758 0 -1.855891 41.95948

. mata: matrixmarkdown(“e(V)”)

price weight 0b.foreign 1.foreign _cons
3.70e-08 -1.23e-07 0 -.0001344 .0001827

The table’s style is modified by a stylesheet (a bit of HTML code). For tables of numbers, it helps to have something – extra padding or cell borders – to separate the table cells visually.
